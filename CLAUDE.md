# CLAUDE.md

## Repository

Personal macOS dotfiles managed with GNU Stow. All scripts require macOS and use `set -euo pipefail`.

Non-obvious facts:
- `kanata/` is NOT stowed тАФ requires manual setup (see `kanata/kanata.md`)
- Brewfile loading is environment-aware: always `Brewfile`, then `Brewfile.home` or `Brewfile.work` based on `whoami`, then `Brewfile.local` if present, then `Brewfile.cache` if non-empty
- `Brewfile.cache` is auto-generated by `repack` (gitignored, loaded last): before bundling it records mas apps and casks that are installed but not declared in any tracked Brewfile, so `brew bundle --zap` won't uninstall them. It's recomputed every run against the tracked Brewfiles only тАФ promote keepers into `Brewfile.home`/`Brewfile.work` and they drop out of the cache. Passing `--skip-cache` to `repack`/`redot` skips that refresh: the existing cache is still honoured (loaded into the bundle), but anything not in the Brewfiles or that cache is zapped тАФ use it to prune apps that have drifted in since the last run without first capturing them. `--clear-cache` goes further: it lists the cache plus any uncached drift, asks for confirmation, then deletes the cache and runs as `--skip-cache` тАФ zapping every untracked app. `--select-cache` is the interactive middle ground: it lists the untracked apps and lets you multi-select which to prune (uninstall), writes the rest back as the cache, then runs as `--skip-cache` so the deselected apps are zapped тАФ surfaced in the TUI as `dot pack` тЖТ "Review cached apps"
- `homebrew/.homebrew/trust.json` is stow-managed: Homebrew's tap-trust file (`~/.homebrew/trust.json`) is a symlink into this repo. Running `brew trust тАж` writes through it тАФ `git add` the change afterward. Only trust taps that appear in a Brewfile.

## Architecture (`bin/`)

`bin/dot.sh` is the front door: no args opens a menu, `dot <sub>` dispatches. The
engines (`repack.sh`, `restow.sh`, `reenv.sh`) hold the real logic; `redot.sh` is a
thin shim for `dot sync`. The zsh functions in `sh/.zfunctions.sh` map
`redot/repack/restow/reenv` onto `dot` subcommands, so flags still flow through
(`repack --clear-cache` тЖТ `dot pack --clear-cache`).

Two sourced libraries (no top-level side effects):
- `bin/lib/status.sh` тАФ shared data: `detect_environment`, `intent_brewfiles`,
  `compute_untracked`, `stow_packages_list`, and the one-line `status_*` summaries.
  This is the single home for the brew intent/cache logic тАФ don't redefine it in an
  engine.
- `bin/lib/ui.sh` тАФ the ONLY file that knows about `gum`. Every wrapper
  (`ui_menu`/`ui_confirm`/`ui_spin`/`ui_box`/`ui_header`/`print_*`) falls back to
  plain text when `gum` is absent or output isn't a TTY, and the interactive ones
  take a safe default (decline / no selection) rather than hang. This fallback
  contract is what keeps the piped `curl | bash` bootstrap and non-interactive runs
  working тАФ preserve it.

`--plan` convention: each engine accepts `--plan` to print a read-only dry-run
summary and exit without side effects. `dot` renders that before asking to apply.
Keep `--plan` paths strictly side-effect-free (no cache writes, no symlinks).

## Shell Scripts (`bin/`)

All scripts must use this header:

```bash
#!/usr/bin/env bash

set -euo pipefail # Exit on error, undefined vars, and pipeline failures
IFS=$'\n\t'       # Stricter word splitting

# Trap Ctrl-C (SIGINT) and exit gracefully
trap 'echo -e "\nInterrupted. Exiting..."; exit 130' INT

# [Brief title] - one-line description
# Supports:
#   - macOS (via [tool])
#
# Usage:
#   ./script-name.sh [args]
#
# Prerequisites:
#   - [dependency] (via Homebrew)
```
